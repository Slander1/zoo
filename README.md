### Архитектурный паттерн
В проекте используется MVP (Passive view), пример `AnimalView`

### Создание нового Animals
Для создания Animals, мы можем брать уже готовые Movers или добавить новые
Для целлостности данных он сразу связан с данными , которые в него инициализируются из модели Presenter  
Так же мы можем по желанию включать и выключать взаимодействие с объектами и комбинировать их как угодно
Хоть сделать жертво-хищника, который будет, когда встретит хищника - умирать
А когда встретит жертву ее съедать
При желании инициализацию данного конструктора можно без проблем вынести в Inspector

Если мы хотим создать дополнительные Movers или CollisionReact нам вообще ничего не придется изменять в классе животных, кроме как указать их.

Например что бы создать гусеницу, которая прыгает и хищник всего неибходимо написать

`public class Snake : PredatorBase<JumpMover, JumpMoverData, 
DefaultPredatorCollisionController, CollisionControllerDataBase>
{ }`

Если не использовать PredatorBase, то всего лишь необходимо пометить дополнительно 
инетерфейсом что это хищник и переопределить методы хищника
`public class Snake : GenericMovableAnimalBase<JumpMover, JumpMoverData, 
DefaultPredatorCollisionController, CollisionControllerDataBase>, IPredator
{ }`

Если не нужен Generic, то прямое насследование от AnimalBase и полная реализация, все компоненты работают с AnimalBase и интерфейсами

Если мы хотим добавить объекты по типу препядствий - то простое насследование от `IInteractableObjectOnScene` позволяет так же строить взаимодействие с этим объектами
Пример `IWall`

<h2 align="center"> Дополнительно</h2>

### Сцены
SceneCoordinator реализован в максимально простой форме. 
Хотя он формально не требовался в рамках тестового задания, 
я включил его как базовый компонент любой игры: при дальнейшем расширении проекта он позволит легко управлять сценами и ускорить холодный/горячий старт, используя загрузку через промежуточную Init-сцену.
Все взаимодействия компонентов при помощи Unity физики, как и прописано в ТЗ.
Кроме разворота, он сделан через transform для ускорения разработки

Поле создается автоматически , адаптивно под размеры экрана

### .asmdef  
Деление на зоны можно сделать гораздо глубже, проект не сильно связанный
Объекты соприкасаются между собой только через делигаты или events , это легко выносится в какой-нибудь hub/handler
Но по скольку это больше относится к прототипу необходимости в этом 0
